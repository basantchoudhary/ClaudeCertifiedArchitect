🏭 Ops Innovation Portfolio · AI + Automation for Azure Databricks Operations

for an Ops Director · GxP-aware

A dedicated track for making a Databricks operations function more robust, reliable, and self-observing — the kind of platform-Ops org that owns pipelines, clusters, cost, data quality, and incident response. It's built for a regulated (pharma / GxP) environment, so the guiding rule is: deterministic where correctness must be guaranteed, AI where judgment and language help — never the other way round.

🧭 Operating principles (read these first — they shape every project below)
  • Read-only first. Every agent starts as an observer & advisor. It earns write/act permissions only after it's been trusted for weeks — climb the autonomy ladder, don't skip it.
  • Human-in-the-loop for any change. AI proposes; a human approves; a deterministic tool executes. No agent mutates prod, cost, or access on its own.
  • Deterministic guardrails, not prompts, for the rules that must hold. Idle-cluster kill, budget caps, access policy — these are code/policy-as-code, never "please remember to…" in a prompt.
  • Everything is audited & reproducible (GxP). Every recommendation and action logs its inputs, evidence, model version, and the approver — audit-ready by design.
  • Mock-data-first. Build and demo against synthetic system-table / job-run data, so nothing touches real GSK infrastructure until it's validated and signed off.

Deterministic vs AI — how to decide what to build

If the task is…Build it as…Why
A rule that must always hold (kill idle clusters > 30 min, block untagged jobs, enforce a budget cap)Deterministic automationGuaranteed, auditable, cheap, no hallucination surface. An LLM here only adds risk.
Structured detection with clear thresholds (freshness SLA breach, failure-rate spike, cost anomaly)Deterministic + AI summaryDetection is math; AI only writes the human-readable narrative on top.
Correlating messy signals & explaining why (root-cause across logs, config changes, metrics)AI agent (read-only tools)Judgment + language over heterogeneous evidence — exactly where an agent earns its keep.
Answering "how do we handle X?" from runbooks & past incidentsRAGGrounded, cited retrieval — no autonomy needed, low risk, fast MTTR win.
A recommendation a human will apply (right-size a cluster, add a DQ rule, tighten a grant)Hybrid (AI proposes → human approves → deterministic applies)AI's reasoning + a human gate + a deterministic executor = safe change management.
