4.9 · Managed Agents vs. Self-Orchestrated

Who hosts the loop — Anthropic or you?

CONCEPT The one distinction that matters

Everything in Cluster 4 so far assumed you host the agent loop — you call POST /v1/messages, inspect stop_reason, run tools, and append tool_results. That's the self-orchestrated model ("Claude API + tool use"). Managed Agents flip the hosting: Anthropic hosts the agent loop and a per-session container where tools execute. You describe the agent once; Anthropic runs it.

🔑 Awareness-level distinction: who hosts the loop and tool execution? Self-orchestrated = you. Managed Agents = Anthropic, plus a hosted per-session container.

CONCEPT The Managed Agents shape

Conceptually there are two objects:

  • Agent — a persisted, versioned object that holds the configuration: model, system prompt, and tools live on the Agent. You create it once; it's referenced by id.
  • Session — a run that references an Agent by id. Each session gets its own per-session container where tools execute. The agent loop runs inside Anthropic's infrastructure.
⚠️ Keep this conceptual for the exam. Know the shape — persisted versioned Agent → Sessions that reference it, hosted loop + per-session container — not the full API surface.

TRADE-OFFS Control vs. managed convenience

✅ Self-orchestrated

  • Full control over the loop and tool execution
  • Runs on your infrastructure, your environment
  • Maximum flexibility for custom orchestration

✅ Managed Agents

  • Anthropic hosts the loop + per-session container
  • Config lives on a persisted, versioned Agent
  • Less to build and operate yourself

EXAM TRAPS

Trap: Confusing who hosts the loop. → In Managed Agents, Anthropic hosts the loop and the per-session container. With "Claude API + tool use," you host the loop and run the tools.
Trap: Thinking a Session carries the model/system/tools. → Those live on the persisted, versioned Agent. A Session just references the Agent by id.
Trap: Assuming Managed Agents means no tool sandbox. → Tools execute in an Anthropic-hosted per-session container, not on your own machines.

PRACTICE

Q1 · In one sentence, what's the core difference between Managed Agents and "Claude API + tool use"?

In Managed Agents, Anthropic hosts the agent loop and a per-session container where tools run; with "Claude API + tool use," you host the loop and execute the tools yourself.

Q2 · Where do a Managed Agent's model, system prompt, and tools live?

On the persisted, versioned Agent object. You create the Agent once, then create Sessions that reference it by id to run it.

Q3 · A team wants Anthropic to run the loop and provide a container for tool execution. Which option?

Managed Agents. That's exactly the hosted-loop + per-session-container model, versus self-hosting the loop with the plain Claude API + tool use.
