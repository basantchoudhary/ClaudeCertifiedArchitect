🔁 The Agentic Loop

flowchart TD
    U([User request]) --> B["Build messages array
system + FULL history
API is stateless — resend every call"]
    B --> C["client.messages.create(model, tools, messages)"]
    C --> D{"response.stop_reason ?"}
    D -->|"end_turn ✅"| E([Return text to user · DONE])
    D -->|"tool_use 🛠"| F["Append WHOLE response.content
then run tool → tool_result"]
    F --> B
    D -->|"pause_turn ⏸"| G["Re-send assistant turn as-is
server resumes · no Continue msg"]
    G --> C
    D -->|"max_tokens ✂"| H["Output truncated
raise max_tokens / stream / continue"]
    H --> B
    D -->|"refusal 🚫"| I([Surface the decline
don't retry same prompt])
    classDef done fill:#d4edda,stroke:#28a745,color:#155724;
    classDef warn fill:#fff3cd,stroke:#ffc107,color:#856404;
    classDef stop fill:#f8d7da,stroke:#dc3545,color:#721c24;
    class E done;
    class H,G warn;
    class I stop;
🧠 The two things people forget: the API is stateless (you resend the full history every call), and you must append the whole response.content (with its tool_use block), not just the .text.

⚠️ Six values total — the "4-value" list in most guides is incomplete. max_tokens means the output was cut off, not "conversation too long."

🧩 The tool-result contract (why you get a 400)

sequenceDiagram
    participant H as Your harness
    participant C as Claude (Messages API)
    H->>C: user turn — weather in Paris?
    C-->>H: assistant turn — tool_use (id = toolu_A)
    Note over H: run get_weather(Paris) → 18C sunny
    H->>C: user turn — tool_result (tool_use_id = toolu_A) ✅ ids match
    C-->>H: assistant turn — It's 18C and sunny. (end_turn)
    Note over H,C: Rules: results in a USER turn · ids MUST match · roles alternate
Break any → HTTP 400. Tool errored? add is_error = true

Golden rule: pick the simplest tier that works. The exam punishes over-engineering.
