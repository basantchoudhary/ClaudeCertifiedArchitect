5. DOMAIN BREAKDOWN & KEY CONCEPTS

Domain 1: Agentic Architecture & Orchestration (27%)

The Agentic Loop (Most Important Concept)

User Request
    |
Claude Agent (with system prompt + tools)
    |
Check stop_reason:
  - "end_turn" -> respond to user, done
  - "tool_use" -> extract tool calls, execute
  - "max_tokens" -> context overflow, truncate/summarize
    |
If tool_use: Execute tool, get result
    |
Pass result back to Claude as new message
    |
Repeat until stop_reason = "end_turn"

Stop_reason Values (Memorize These)

  • end_turn: Claude finished responding, ready to return to user
  • tool_use: Claude called a tool, waiting for result
  • max_tokens: Claude hit token limit, response incomplete
  • stop_sequence: Claude hit a stop sequence (rare)

Multi-Agent Patterns

  • Coordinator-Subagent: One coordinator decides, multiple specialists execute
  • Hub-and-Spoke: Central agent with many tools (simpler than multi-agent)
  • Sequential Pipeline: Agents run one after another (research -> outline -> draft -> edit)
Exam Strategy: Questions about orchestration, stop_reason handling, and multi-agent patterns make up ~27% of exam. Practice these scenarios heavily.

Domain 2: Claude Code Configuration & Workflows (20%)

CLAUDE.md Hierarchy

Project Root/
|-- CLAUDE.md          (applies everywhere)
|-- subdir1/
|   |-- CLAUDE.md      (applies only to subdir1)
|-- subdir2/
    |-- CLAUDE.md      (applies only to subdir2)

Key Components

  • Skills: Reusable markdown instructions (stored in .claude/skills/)
  • Commands: Custom slash commands (/review, /test, etc.)
  • Hooks: Lifecycle events (before_command, after_command, on_error)
  • Subagents: Isolated agents with restricted tools

Exam Tip

Questions often ask: "How to configure CLAUDE.md for a team?" or "What's the scope of this rule?" Study path-scoping and frontmatter syntax.

Domain 3: Prompt Engineering & Structured Outputs (20%)

System Prompt Design

Be specific. Bad example: "You're a helpful assistant." Good example: "You are a customer support specialist with 10 years experience. Keep responses under 150 words. If billing is involved, escalate to senior agent."

Structured Output (JSON)

Constrain Claude to return valid JSON. Example schema:

{
  "resolution": "string",
  "confidence": "number (0-1)",
  "escalate": "boolean",
  "reason": "string"
}

Few-Shot Examples

Show Claude 2–5 examples of input -> desired output. More effective than lengthy instructions for consistent output.

Domain 4: Tool Design & MCP Integration (18%)

Tool Definition Quality

The description is what Claude reads. A clear description > a clever function name.

GOOD:
"Execute read-only SELECT queries. Use for: finding orders,
checking account status. DO NOT use for: INSERT, UPDATE, DELETE.
Always include WHERE clause."

BAD:
"Query database"

MCP Primitives

  • Tools: Callable functions
  • Resources: Data Claude can read (docs, config)
  • Prompts: Reusable instructions
  • Sampling: LLM-powered logic in MCP server

Domain 5: Context Management & Reliability (15%)

Token Budgeting

Each message token costs money. Conversation history grows. At some point, you hit the token limit. Solution: prune old messages, summarize.

Prompt Caching

Cache system prompt + reference docs. Reuse across requests (~90% cheaper for cached tokens).

"Lost in the Middle" Problem

Claude pays less attention to middle of long prompts. Solution: put important info at start/end, verbose content in middle.

Error Recovery

  • Retry: If transient failure (timeout), retry with backoff
  • Fallback: If tool fails, use alternative or skip
  • Escalation: If repeated failure, escalate to human
