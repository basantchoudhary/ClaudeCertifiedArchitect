🤖 What's Inside an LLM — Transformers & Attention

Tokens → embeddings → attention → next token, on repeat

BIG IDEA An LLM is a next-token predictor

Strip away the mystique and a large language model does one thing: given a sequence of tokens, it predicts the probability of the next token. It picks one, appends it, and predicts again — over and over. That loop is called autoregressive generation. Everything impressive an LLM does — answering, coding, reasoning — emerges from doing this next-token guess extremely well, at massive scale.

🔑 The architecture that made this work well is the Transformer (2017, "Attention Is All You Need"). The GPT in ChatGPT literally stands for Generative Pre-trained Transformer. Claude, Gemini, and Llama are all Transformers too.

  1. Tokenize — split text into tokens (see tokens).
  2. Embed — turn each token into a vector of numbers that captures meaning, plus its position in the sequence.
  3. Transformer blocks — stacked layers (dozens to 100+) that repeatedly mix information between tokens (attention) and process each token (feed-forward).
  4. Logits — the final layer outputs a score for every token in the vocabulary.
  5. Sample — convert scores to probabilities (softmax) and pick one — how "random" depends on temperature. Append it and go again.

ATTENTION The key trick: tokens look at each other

Attention is what lets the model understand context. When processing a word, the model attends to the other words that matter for its meaning — no matter how far away they are. Take:

"The trophy didn't fit in the suitcase because it was too big."

To resolve "it," the model attends back to "trophy" (not "suitcase"). Attention computes, for every token, how much each other token should influence it, and blends them accordingly. Doing this across many layers and many parallel "attention heads" (each learning different relationships — grammar, references, topic) is what builds rich understanding.

This is why the context window matters and why long prompts get pricey: every token attends to every other token, so cost grows quadratically with sequence length. Longer context = a lot more attention math.

SCALE Why "just predicting the next token" is so powerful

To predict the next token well across the entire internet, a model is forced to learn grammar, facts, translation, code, and reasoning patterns — because all of those help it guess better. Push three dials up together — more parameters (the learned weights, the "B" in "70B"), more data, more compute — and new capabilities appear. That's the "scaling" story behind modern LLMs.

✅ What the Transformer gave us

  • Handles long-range context via attention
  • Processes a sequence in parallel (fast to train)
  • Scales smoothly with size & data

⚠️ The costs

  • Attention is quadratic in length → long context is expensive
  • Huge models need huge compute to train & run
  • It predicts plausible text — hence hallucination

TAKEAWAYS

Remember: an LLM generates one token at a time, feeding its own output back in — that's why output is the slow, costly part.
Remember: attention is how the model uses context; it's also why doubling your prompt length more-than-doubles the work.
Remember: the model has no database of facts — knowledge is baked into weights as statistical patterns, which is why it can be confidently wrong (ground it with RAG).
