1. EXAM OVERVIEW

Certification: Claude Certified Architect – Professional (CCAR-P)
Format: 90–120 scenario-based questions
Duration: 180–210 minutes (3–3.5 hours)
Pass Score: Higher than CCA-F (estimated ~75%+)
Cost: $175 USD
Prerequisites: CCA-F passed OR 12+ months production experience
Launch Date: July 2026

What CCAR-P Actually Tests

CCA-F: "Can you build a Claude system?"
CCAR-P: "Can you design the right Claude system for a complex business problem?"

The shift is from technical capabilities to architectural judgment.

Example Question Progression

  • CCA-F: "How does an agentic loop work?"
  • CCAR-P: "A client processes 10,000 requests/day with <1s latency SLA, $2k/month budget, and HIPAA compliance. Design the agent architecture. What trade-offs?"

6. THE SIX DOMAINS IN DEPTH

Domain 1: Solution Architecture & Business Requirements (25%)

The Core Concept: Translation

Business problem + Constraints -> Claude Architecture

Constraints You'll See

  • Latency SLA: <100ms (impossible), <500ms (optimize hard), <1s (standard), <5s (easy)
  • Cost Budget: $100/month (batch only), $1k/month (optimize), $10k/month (scale freely)
  • Scale: 10 req/day (simple), 1000/day (cache needed), 100k/day (distributed, parallel)
  • Compliance: HIPAA (encrypt, audit log), GDPR (deletion rights), SOC2 (audit trail)
  • Reliability: 99% (simple), 99.9% (redundancy + fallbacks)

Architectural Patterns

  • Simple Agent: User -> Claude with tools -> Response
  • RAG: Query -> Retrieve -> Claude with context -> Answer + sources
  • Multi-Agent: Coordinator -> Specialists -> Merge -> Response
  • Batch Processing: Queue requests -> Batch API -> Process overnight -> Results
Exam Example: "Legal document review system. 1000 contracts/day. <2s latency. $5k/month budget. GDPR compliance. Design architecture, calculate costs, justify model choice."

Domain 2: Advanced Agentic Systems (20%)

When to Use Multi-Agent

Simple support ticket Single Agent + Tools
Complex workflow (5+ steps) Multi-Agent
Parallel work (research + draft simultaneously) Multi-Agent
Specialized expertise needed Multi-Agent
Cost-sensitive Single Agent + Tools

Autonomy Levels

  • Supervised: Agent suggests, human approves (safe, slow)
  • Semi-Autonomous: Agent acts, human can override (balanced)
  • Fully Autonomous: Agent acts freely (fast, risky)

Failure Modes

  • Cascade failure: Agent1 fails -> cascades to Agent2, Agent3. Solution: circuit breaker
  • Lost state: Subagent doesn't know context. Solution: pass full state explicitly
  • Timeout chain: Sequential agents = additive latency. Solution: parallelize independent agents

Domain 3: Tool Design & MCP Integration (15%)

Security-First Design

  • Input Validation: Never trust Claude's output. Validate type, range, format.
  • Rate Limiting: Per-user (100 queries/min), per-tool (10 concurrent)
  • Access Control: Who can call this tool? What data can they access?
  • Audit Logging: Log every call (who, what, when, result). Tamper-proof storage.
  • Error Handling: Never expose internal errors. Generic message: "Connection timeout, retry later"

Tool Description Quality

The description is what Claude reads. Make it detailed, not clever.

MCP Primitives

  • Tools: Callable functions (what Claude uses most)
  • Resources: Data Claude can read (docs, config, references)
  • Prompts: Reusable instructions (checklists, guidelines)
  • Sampling: LLM-powered logic in MCP server (advanced, rare)

Domain 4: Prompt Engineering & Context Management (15%)

Retrieval Strategy Maturity

  • Level 1 (Simple): Query -> embed -> vector search -> Claude. Quality: 60–70%
  • Level 2 (Hybrid): Semantic + keyword search. Quality: 75–85%
  • Level 3 (Reranking): Semantic -> rerank -> Claude. Quality: 80–90%
  • Level 4 (Time-Aware): Filter by date, then retrieve. Quality: 85–95%
  • Level 5 (Graph-Based): Entity relationships + semantic. Quality: 90%+ (complex)

"Lost in the Middle" Problem

Claude pays attention to start/end of long prompts, ignores middle. Solution: key info at start/end.

Prompt Caching ROI

Setup: 5 min to warm cache = $0.01. Savings per request: $0.01 (90% discount). Break-even: 1 request.

Domain 5: Security, Compliance & Governance (15%)

Threat Model (OWASP)

  • Data breach: Mitigation: encrypt at rest + in transit
  • Unauthorized access: Mitigation: authentication + authorization (RBAC)
  • Prompt injection: Mitigation: validate inputs, don't dynamically construct prompts
  • Model drift: Mitigation: monitor eval metrics, re-evaluate regularly
  • PII leakage: Mitigation: redact before storing, audit logs

Compliance Frameworks

  • SOC2: Access controls, change mgmt, incident response, audit logging
  • HIPAA: Encryption, audit logging, PHI isolation, data retention policies
  • GDPR: Right to access, right to deletion, data minimization, purpose limitation

Domain 6: Evaluation, Monitoring & QA (10%)

Production Metrics

  • Latency: p50, p95, p99 (95th percentile matters most)
  • Cost: $ per request, trending up/down?
  • Error Rate: % of requests failed. Target: <0.1%
  • Quality: Accuracy, user satisfaction, relevance (business metric)

Evaluation Framework

  • Automated (daily): ROUGE, semantic similarity, fact-checking. Fast, cheaper.
  • Human (weekly): Sample 100 requests, experts rate 1–5. Expensive but definitive.
  • Model-Based (daily): Use another Claude to score. Cheap, correlate with human eval.

CI/CD for LLM Systems

Code change -> unit tests -> eval suite (blocks if quality drops) -> canary (5%) -> 25% -> 100%

7. ENTERPRISE ARCHITECTURE PATTERNS

Pattern 1: Real-Time Document Processing

Constraint: 1000 contracts/day, <2s per contract, HIPAA compliant
Architecture: Parallel extraction agents (entity, risk, recommendation)
Model: Sonnet (good extraction, cost balance)
Output: JSON schema (structured extraction)
Compliance: Encrypt at rest, redact PII, audit log all access

Pattern 2: Autonomous Customer Support at Scale

Constraint: 10,000 tickets/day, <3s response, complex escalation
Architecture: Coordinator -> specialist agents (refund, technical, billing)
Autonomy: Semi-autonomous (auto-refund <$50, escalate >$100)
Escalation: Confidence-based + rule-based (payment? escalate)
Cost: Multi-agent costs more but handles 10x volume

Pattern 3: Evaluation-Driven RAG

Goal: 95% answer accuracy, continuous improvement
Pipeline: Eval set -> identify failures -> adjust (chunking, prompts, docs) -> A/B test -> deploy
Success: Weekly human eval, trend monitoring, auto-alert if quality drops
